4.2 技能获取

反向操作 4.1 技能分享：把别人的 SKILL.md 装到自己 Studio。本节描述四种获取路径及相关决策点。

四种获取路径​

全部入口都在 技能工坊。

路径 1：内置技能中心（最快）​

技能工坊 → 技能中心 → catalog → 浏览 D-Robotics 出品的官方技能目录（45+ 个）。按分类组织：

  • 核心操作（core/）：设备诊断、硬件知识、OpenClaw 协作
  • 板型专属（boards/）：RDK X5 应用、X5 AI 检测
  • 文档与搜索（docs/）：RDK 官方文档、TROS 文档
  • 通用工具（tools/）：浏览器自动化、搜索引擎
  • 可选扩展（optional/）：Token 用量报告等

点击 安装到本地 或 部署到板端 即可。

路径 2：ClawHub 社区搜索​

技能工坊 → 技能中心 → ClawHub 社区 → 输入关键词搜索：

操作怎么做
关键词搜索顶部搜索框
预览技能点技能名 → 看 SKILL.md 全文与 frontmatter
一键安装安装到本地 或 部署到板端
批量安装勾选多个技能 → 批量装

ClawHub 默认走国内镜像。如果搜不到目标，确认网络可达 ClawHub 域名。

路径 3：从 URL 导入​

把任意外部内容转化为技能：

URL 类型自动识别规则
GitHub 仓库URL 含 github.com/...
NodeHub 案例URL 含 developer.d-robotics.cc/nodehub
普通网页 / 博客默认

操作步骤：

  1. 技能工坊 → 技能中心 → 链接
  2. 粘贴 URL，Studio 自动识别来源
  3. 填写目标描述："我想让 AI 学会 X"
  4. 选择适用设备型号（X3 / X5 / S100 / 通用）
  5. AI 自动抓取并转化为 SKILL.md
  6. 预览 → 调整 → 安装

路径 4：手动放文件​

最低门槛——同事或群里发来一个 SKILL.md：

  1. 保存到本机工作区 skills/<name>/SKILL.md
  2. Studio 自动扫描发现，无需重启
  3. 在 已安装 列表能看到它

安装位置的选择​

选项装到哪谁能用
安装到本地Studio 进程内PC 端 D-Moss Agent
部署到板端通过 OpenClaw 同步到板端 OpenClaw 工作区板端 Agent

如果希望板端常驻 Agent 也能使用该技能（例如长期监控、断网自治），必须选择"部署到板端"。仅安装到本地的技能板端 Agent 看不到。

装多少合适​

数量影响
小于 10无感
10~30AI 选技能耗时增加（毫秒级）
大于 50上下文膨胀，AI 决策可能受干扰

不要"装一堆备用"。用什么装什么，没用就在 已安装 列表卸载。

验证已装的技能​

AI Dock 中输入 /skills → 列出当前会话注入的技能 ID 与简要说明。

注入是按需动态加载的——不是所有装过的技能都同时在上下文。/skills 显示的是"目前这个会话已激活的"。

让 AI 帮你找技能​

不知道哪个技能能解决你的问题？直接对 AI Dock 说"有没有让我能调试 USB 摄像头的技能？" Agent 会在内置目录和 ClawHub 里搜索，列出几个候选并简要说明各自的能力，你选一个让它装上。

常见获取问题​

现象解决
ClawHub 搜索无结果换更通用的关键词；确认网络可达 ClawHub 域名
URL 导入失败URL 不可达、内容太长、格式不识别；可手工转 SKILL.md 走路径 4
装上后 AI 没使用检查 SKILL.md 的 trigger 与你的实际场景是否匹配；必要时修改 trigger 后重启 Studio
技能太多导致 AI 决策乱卸载不用的技能，见 3.11.5 触发匹配机制 的优化建议
