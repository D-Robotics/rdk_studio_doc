# 3.3.1 多标签 SSH 会话

远程终端支持同时打开多个独立的 SSH 标签，每个标签拥有独立的会话与 PTY 进程，互不干扰。这一设计让"一个标签跑长任务、另一个标签查状态"的常见工作流成为标准用法。

## 标签的独立性

每个标签是一次独立的 SSH 会话，意味着：

- 在标签 A 中跑的环境变量、cd 路径、shell 函数不会影响标签 B
- 标签 A 的命令被中断（Ctrl+C）不影响标签 B 中正在运行的命令
- 关闭某个标签只断开该标签的 SSH 会话，板端如果有用 `nohup` 起的进程会继续运行

## AI 联动：透明 Agent

AI Agent 通过 `device_exec` 工具调用的命令同样会在远程终端中实时显示。这意味着：

| 触发者 | 命令显示位置 |
|---|---|
| 开发者在终端手动输入 | 当前标签 |
| AI Agent 通过工具调用 | 同样在当前标签实时显示 |

这一设计让 Agent 的每一步操作都对开发者可见，避免"AI 在背后偷偷做事"。如果 Agent 选择了不期望的命令，开发者可以按 Ctrl+C 直接在终端中中断（同时 AI Dock 中的 Agent 任务也会同步取消）。

## 切换设备时的标签行为

切换当前激活设备时，远程终端默认在新标签中打开新设备的会话，旧设备的标签保留。这种设计避免"切设备时把已经跑起来的长任务断掉"。如果不需要保留旧标签，可以手动关闭。

## 标准 Shell 快捷键

终端完整支持标准 Shell 快捷键：

| 快捷键 | 作用 |
|---|---|
| Ctrl + C | 中断当前命令 |
| Ctrl + D | 退出 shell（关闭标签） |
| Ctrl + L | 清屏（焦点在终端时） |
| Tab | 自动补全 |
| ↑ / ↓ | 历史命令 |
| Ctrl + R | 反向搜索历史 |
| Ctrl + W | 删除前一个词 |
| Ctrl + U | 清空整行 |


## 复制与粘贴

| 平台 | 复制 | 粘贴 |
|---|---|---|
| Windows / Linux | 选中文本自动复制（避免按 Ctrl + C 中断命令） | Ctrl + Shift + V 或鼠标中键 |
| macOS | 选中 + Cmd + C | Cmd + V |
