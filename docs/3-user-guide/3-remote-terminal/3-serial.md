# 3.3.3 串口连接

远程终端 tab 同时支持串口连接（无需 SSH），与 SSH 会话在同一界面内统一管理。串口连接是板端无法 SSH 时的救火方式，详细的接线、配置、救火示范见 [2.3.3 通过串口接入](../../2-quick-start/3-connect-device/3-serial.md)。

## 串口与 SSH 的统一入口

Studio 的远程终端不区分 SSH 与串口——两者都通过同一个 *新建标签* 入口创建：

| 选项 | 行为 |
|---|---|
| 选择 *SSH 会话* | 建立 SSH 连接到当前激活设备 |
| 选择 *串口连接* | 弹出串口配置对话框，选择串口设备与波特率 |

这种设计让开发者在排障场景中无缝切换：板端启动失败时通过串口诊断，启动成功后切到 SSH 继续完整开发流程。

## 串口能力的边界

串口只提供终端能力，不支持：

- 文件传输
- 远程桌面
- 远程 IDE
- OpenClaw 部署（需要网络）

如果排障过程中需要传文件或操作图形界面，必须再用 SSH 方式连接同一台设备。Studio 允许同时通过 SSH 与串口访问同一设备，两条通道独立工作。

## 控制台日志面板

远程终端 tab 边上还有一个"控制台"面板，显示的是 **Studio 后端**的运行日志（不是板端日志）。这一面板用于诊断 Studio 自身的问题，例如 SSH 会话建立失败、模型 API 调用异常等。日常使用中通常不需要查看这一面板，仅在 Studio 自身行为异常时打开。
