# 3.13.2 微信通道

微信通道通过绑定开发者的微信个人账号实现 AI 对话接入。请注意：这是绑你的**个人微信账号**，不是企业微信。绑定后开发者可以在微信中给 Bot 发消息、查询板端状态、远程触发任务。

## 绑定步骤

| 步骤 | 操作 |
|---|---|
| 1 | *配置中心 → 多通道接入 → 微信* → 点击 *绑定微信* |
| 2 | Studio 弹出二维码 |
| 3 | 用微信扫码 |
| 4 | 在微信中收到 Bot 的"绑定成功"消息 |
| 5 | 之后给 Bot 发消息即可与 AI 对话 |

绑定后微信中会出现一个新的联系人（或群机器人，取决于实现方式），与该联系人对话即与 AI 对话。

## 管理操作

| 操作 | 路径 |
|---|---|
| 查看已绑定账号 | *微信 → 用户列表* |
| 移除某个绑定 | *用户列表 → 选账号 → 移除* |
| 重启微信通道 | *微信 → 通道控制 → 重启* |

## 微信场景的限制

| 限制 | 影响 |
|---|---|
| 个人微信 | 同时只能绑定**少量账号**（具体上限取决于微信策略） |
| 高风险操作 | 同样会要求二次确认，与飞书通道一致 |
| 富文本展示 | 微信对话框样式有限，长输出会被截断或分段 |
| 文件传输 | 受微信 API 限制，大文件需用其他方式（如 SSH 直传） |
| 账号风险 | 频繁的自动化消息可能触发微信封号策略，谨慎使用 |

## 适用场景

| 场景 | 推荐 |
|---|---|
| 出差或离开办公室时查询板端状态 | 推荐 |
| 在微信群中由团队成员操作板端 | 不推荐（建议用飞书企业 Bot） |
| 7×24 自动化机器人接入 | 不推荐（账号风险，建议使用 Bot 平台或飞书） |

## 部署在板端 OpenClaw

如果希望"PC 关机后微信消息也能被处理"，可以把微信通道部署在板端 OpenClaw：

1. 进入 *OpenClaw → 配置 → 微信通道*
2. 配置或扫码绑定
3. 板端 OpenClaw 接管微信消息处理
4. PC 关机不影响——板端只要有电、有网就能处理消息

板端绑定与 PC 端绑定可以独立存在，但不建议同时绑定同一个微信账号（可能引起消息处理冲突）。
