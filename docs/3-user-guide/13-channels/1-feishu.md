# 3.13.1 飞书通道

飞书通道把 RDK Studio 的 AI 对话能力接入企业飞书机器人，团队成员可以：

- 在飞书群中 @Bot 让 AI 帮你操作板端
- 在私聊中 1:1 与 AI 对话
- AI 的所有响应都在飞书可见，任务流可追溯

## 配置项

在 *配置中心 → 多通道接入 → 飞书* 配置以下字段：

| 字段 | 必填 | 来源 |
|---|---|---|
| App ID | 是 | 飞书开发者后台 → 你的应用 → 凭证与基础信息 |
| App Secret | 是 | 同上 |
| Encrypt Key | 否 | 飞书后台 → 事件订阅 |
| Verification Token | 否 | 同上 |

App ID 和 App Secret 是飞书自建应用的鉴权凭据。Encrypt Key 用于加密飞书推送的事件，Verification Token 用于验证事件来源——两者均可选，未配置时使用默认安全策略。

## 创建飞书自建应用

如果团队还没有飞书自建应用，需要先创建：

1. 进入飞书开发者后台
2. 创建新的"自建应用"，填写应用名称、图标、描述
3. 在 *权限管理* 中开启所需权限：消息读取、发送消息、群组信息等
4. 在 *事件订阅* 中订阅"接收消息"事件，填写回调 URL（指向 Studio 实例的对外地址）
5. 在 *版本发布* 中提交应用，等待企业管理员审批
6. 审批通过后获取 App ID 和 App Secret

详细流程请查阅飞书开放平台官方文档。

## 三种私聊策略

| 模式 | 行为 | 适用场景 |
|---|---|---|
| 白名单模式 | 仅指定的飞书用户能与 Bot 私聊 | 生产环境的强烈推荐 |
| 审批模式 | 任何人发消息，需要管理员审批一次后才能继续使用 | 团队内部、希望管控但不太严格 |
| 开放模式 | 任何能看到 Bot 的飞书用户都能直接使用 | 测试环境、明确无敏感操作的场景 |

**生产环境务必使用白名单模式。** 开放模式 = 任何人都能让 AI 在你的板端跑命令，风险极高。

## 用户管理

| 操作 | 路径 |
|---|---|
| 查看已绑定用户 | *飞书 → 用户管理* |
| 审批待处理的配对请求 | *飞书 → 待审批* |
| 移除某个用户 | *用户管理 → 选用户 → 移除* |
| 启动 / 停止整个飞书通道 | *飞书 → 通道开关* |

## Bot 在飞书中的能力

Bot 能干的事情与 AI Dock 几乎相同：

- 查看设备状态："查看 RDK-X5-工位1 的 BPU 占用"
- 文件传输："把 /tmp/log.txt 发给我"（Bot 把文件作为附件发到飞书）
- 命令执行："重启 ROS 节点 camera_publisher"
- 调用所有已安装的技能

在飞书中调用高风险命令（如 `rm -rf`、`kill`、`dd`）时，Bot 会强制要求用户在飞书中二次确认（即使 Studio 内默认放行）。这是因为飞书是外部通道，安全策略更严格。
