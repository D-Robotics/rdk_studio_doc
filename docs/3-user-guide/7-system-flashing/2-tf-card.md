# 3.7.2 TF 卡烧录

![烧录向导 · 第 2 步选择镜像：列出 RDKOS 3.4.1 Desktop/Server、3.3.3 Desktop/Server 等官方镜像，右侧可上传本机镜像文件](http://rdk-doc.oss-cn-beijing.aliyuncs.com/doc/img/rdk_studio/zh/flash-wizard-select-image.png)

TF 卡烧录是 RDK X3 与 RDK X5 最常用的烧录方式。Studio 内置烧录器封装了底层的 dd 操作，提供原生磁盘扫描、自动权限提升、进度展示等能力。

## 操作步骤

1. 把 TF 卡插入 PC 读卡器
2. 打开桌面客户端，进入 *系统烧录* tab，选择 *TF 卡烧录*
3. 选择目标板型（RDK X3 或 RDK X5）
4. 选择镜像版本（详见下方"镜像选择"）
5. 在目标 TF 卡列表中选中正确的卡（**仔细确认！烧录会清空所有数据**）
6. 选择烧录模式：balanced 或 turbo
7. 点击 *开始烧录*
8. 等待 5~15 分钟（视卡速度与镜像大小）

烧录中可以切换到其他 tab 查看 AI 对话或运行命令，但不能拔出 TF 卡或让 PC 进入睡眠。

## 烧录模式对比

| 模式 | 速度 | 适用场景 |
|---|---|---|
| balanced（默认） | 中等 | 兼容性最好，适合大多数 TF 卡 |
| turbo | 快 1.5~2 倍 | 仅适合高质量 TF 卡（SanDisk Extreme Pro 等 A2 等级） |

如果不确定 TF 卡质量，使用 balanced 模式更稳妥。劣质卡用 turbo 模式可能在中途校验失败。

## 镜像选择

Studio 烧录页默认列出 D-Robotics 官方推荐镜像：

| 板型 | 推荐镜像 | 含桌面 |
|---|---|---|
| RDK X3 | RDKOS 3.0.x | Desktop（含桌面）/ Server（纯命令行） |
| RDK X5 | RDKOS 3.x（最新版） | Desktop / Server |

如果不确定选哪个版本，**选最新版**通常是正确的。需要回退到旧版本的场景较少，仅在特定项目要求时才使用。

## 镜像格式支持

| 格式 | 说明 |
|---|---|
| `.img` | 原始磁盘镜像 |
| `.img.xz` | xz 压缩，体积更小，下载更快 |
| `.img.gz` | gzip 压缩，常见的 Linux 发行版镜像格式 |

Studio 在烧录时自动解压压缩镜像，开发者无需手动解压。

## 上传本地镜像

如果使用自定义镜像（例如团队定制版本）：

1. 进入 *系统烧录 → 本地镜像*
2. 选择 PC 上的 `.img` / `.img.xz` / `.img.gz` 文件
3. 后续步骤与使用内置镜像一致

## 常见问题

| 现象 | 排查 |
|---|---|
| TF 卡未识别 | 换读卡器、换 USB 口；TF 卡可能损坏 |
| 写入失败或校验失败 | TF 卡可能有坏块，格式化重试或换卡 |
| 烧录完成插入板端无法启动 | 镜像版本与板型不匹配，重新选择官方推荐版本 |

完整的故障排查见 5.x 常见问题中的烧录相关章节。
