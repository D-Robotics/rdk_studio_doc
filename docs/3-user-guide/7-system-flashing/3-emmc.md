# 3.7.3 eMMC 烧录与备份

RDK X5 带 eMMC 版本可以把系统烧到板载 eMMC，不依赖外部 TF 卡。eMMC 烧录适合生产部署，但烧录前强烈建议先做 eMMC 备份——eMMC 烧录会清空所有数据，备份是回退到原状态的唯一途径。

## eMMC 烧录步骤

1. 板端进入烧录模式（按板手册操作 boot 跳线）
2. 用 USB 数据线将板端连接到 PC
3. 在桌面客户端进入 *系统烧录 → eMMC 烧录*
4. Studio 自动识别板端，开始烧录
5. 等待烧录完成（约 10~30 分钟，视镜像大小）

烧录完成后，板端会自动重启进入新系统。

## eMMC 备份

强烈建议在每次 eMMC 烧录前先做一次备份。备份流程：

1. 进入 *系统烧录 → eMMC 备份*
2. Studio 检测板端 eMMC 状态
3. 启动备份，实时显示进度
4. 备份完成后下载到 PC 本地（保存为 `.img` 文件）

备份镜像可以反向还原：在 *本地镜像* 中选择该备份文件，按 eMMC 烧录流程即可恢复到备份时的状态。这种"备份 → 升级 → 必要时回退"的工作流在生产环境中尤其重要。

## 烧录模式切换

板端进入烧录模式的方式取决于具体板的硬件设计：

- 跳线或拨码：参考板的硬件手册
- USB OTG：部分板支持通过 USB OTG 接口直接进入烧录模式

烧录完成后需要还原到正常 boot 模式（恢复跳线或断开 OTG），重启板端从 eMMC 启动。

## 与 TF 卡烧录的对比

| 维度 | TF 卡烧录 | eMMC 烧录 |
|---|---|---|
| 操作便利性 | 高（PC 上直接烧录 TF 卡） | 中（需要切换板端 boot 模式） |
| 烧录速度 | 快（5~15 分钟） | 较慢（10~30 分钟） |
| 升级便利性 | 高（直接换 TF 卡） | 中（需要重新进入烧录模式） |
| 可靠性 | 中（TF 卡可能寿命到期） | 高（eMMC 寿命远长于 TF 卡） |
| 适合场景 | 开发、测试、频繁切换镜像 | 生产部署、长期运行 |
