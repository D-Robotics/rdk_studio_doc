# 3.7.1 烧录介质对比

RDK Studio 支持三种烧录介质，分别面向不同板型与不同使用场景。本节给出对比表与选择建议。

## 三种介质对照

| 介质 | 适用板型 | 推荐场景 | 介质成本 |
|---|---|---|---|
| TF 卡（SD 卡） | RDK X3、RDK X5 | 开发调试、需要快速切换镜像版本 | 低（一张 32 GB SanDisk Extreme 约 50 元） |
| eMMC（板载） | RDK X5 带 eMMC 版本 | 生产部署、不依赖外部存储 | 板端固定，无需购买 |
| RDK S100 xburn | RDK S100 | RDK S100 唯一的烧录方式 | 板端固定 |

## 选择建议

| 你的板型与目标 | 推荐介质 |
|---|---|
| RDK X3 任何场景 | TF 卡 |
| RDK X5（不带 eMMC） | TF 卡 |
| RDK X5（带 eMMC）开发阶段 | TF 卡（便于切换镜像） |
| RDK X5（带 eMMC）量产部署 | eMMC（无外部介质，更可靠） |
| RDK S100 任何场景 | xburn 工具 |

## TF 卡选购建议

不是所有 TF 卡都适合做 RDK 系统盘。建议选择：

- 容量 ≥ 16 GB（推荐 32 GB 或 64 GB）
- 速度等级 Class 10 或更高（推荐 A1 / A2 应用性能等级）
- 来源可靠的品牌（SanDisk、Samsung、Kingston）

劣质 TF 卡常见问题：

- 实际容量虚标，烧录中途失败
- 写入速度慢，烧录耗时数十分钟
- 寿命短，使用 1~2 个月后频繁出现读写错误

## 烧录前的准备

无论使用哪种介质，烧录前都建议：

- 确认目标介质上的数据已备份（烧录会清空所有数据）
- 板端断电，避免烧录中误操作触发板启动
- PC 接好稳定电源（笔记本接电源适配器，避免烧录中休眠）
