# 3.4.3 路径访问控制

部分敏感路径（如 `/sys`、`/proc`）的写入会受到 Studio 安全策略限制。这一机制是为了避免 AI Agent 误操作搞坏板端系统，例如在错误的内核接口写入数据导致系统崩溃。

## 受限的路径

| 路径 | 受限原因 |
|---|---|
| `/sys/` | Linux 内核接口，写入特定文件可能改变内核行为甚至导致硬件异常 |
| `/proc/` | Linux 进程与内核状态接口，写入可能干扰运行中的进程 |
| `/dev/` | 设备节点，错误写入可能导致设备异常 |
| `/boot/` | 系统启动相关文件，错误修改可能导致系统无法启动 |

读取这些路径不受限制——开发者可以正常浏览、查看内容。仅写入操作受到拦截。

## 例外授权方式

如果开发者确实需要向受限路径写入，有两种方式：

### 方式一：使用远程终端

在 [3.3 远程终端](../3-remote-terminal/index.md) 中手动执行命令：

```bash
echo 1 > /sys/class/leds/red/brightness
```

远程终端不经过文件管理的访问控制层，所有命令直接由板端 shell 执行。

### 方式二：在 AI Dock 中明确授权

在 AI Dock 中描述需求时明确授权，例如：

> "我授权你向 /sys/class/leds 路径写入。请把 LED 红灯打开。"

Agent 收到这种明确授权后会执行写入。这种方式适用于"既想用 AI 自动化，又要操作敏感路径"的场景。

## 拦截行为

当文件管理拦截写入操作时，会显示如下提示：

```
该路径处于受限范围，无法直接通过文件管理写入。
请使用远程终端手动操作，或在 AI Dock 中明确授权。
```

不会出现"看似成功但实际未写入"的歧义状态——拦截发生时会明确告知开发者。
