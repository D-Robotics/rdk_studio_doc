# 3.2.5 斜杠命令

斜杠命令是在 AI Dock 输入框中以 `/` 开头的特殊指令，用于触发 Studio 的内部行为而不是发送给大模型。这类命令不消耗 Token，立即执行。

## 命令清单

| 命令 | 作用 |
|---|---|
| `/help` | 显示所有可用斜杠命令的清单与说明 |
| `/clear` | 清空当前会话的上下文，从空白对话重新开始（保留会话记录） |
| `/sessions` | 列出所有历史会话，可切换到任一会话继续 |
| `/skills` | 列出当前会话已加载的技能 ID 与简要说明 |
| `/model` | 临时切换当前对话使用的模型（仅本会话生效） |
| `/reset` | 深度重置 Agent 状态：清空上下文、重置工具加载状态、重新探测设备 |

## 典型使用场景

### `/clear` vs 新建会话

`/clear` 清空当前会话的上下文但保留会话记录，适用于"在同一个会话里换个话题继续"。新建会话则完全独立，适用于"开始一个全新的任务"。开发者可以根据场景选择。

### `/skills` 验证技能加载

如果开发者怀疑某个技能没被 Agent 自动加载（比如安装了某个技能但 Agent 给的回复看起来没用上），可以输入 `/skills` 查看当前会话实际激活的技能清单。

技能加载是按需的：用户消息中命中 trigger 关键词的技能才会被加载。如果用户的描述与技能的 trigger 不匹配，技能不会被激活。这种情况下可以在 [3.11 技能](../11-skill/index.md) 中调整技能的 trigger 关键词。

### `/model` 临时换模型

某次对话需要更强的模型（例如复杂代码生成），可以临时用 `/model` 切换：

```
/model claude-sonnet-4-20250514
```

切换后立即生效，仅影响本会话，不修改全局默认配置。会话结束或新建后恢复到默认模型。

### `/reset` 处理 Agent 异常

当 Agent 看起来"卡住了"——例如反复调用同一个失败的工具、回复明显与上下文脱节——可以用 `/reset` 深度重置。这会清空对话上下文、重新加载工具、重新探测当前设备。比 `/clear` 更彻底。
