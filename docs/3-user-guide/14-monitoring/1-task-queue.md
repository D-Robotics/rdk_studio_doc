# 3.14.1 任务队列

任务队列位于 RDK Studio 顶栏右侧，把当前进行中的工作台相关任务集中展示。开发者无需切换 tab 就能看到长任务的进度。这一设计参考了浏览器底部的下载列表。

## 顶栏会出现的任务类型

| 类型 | 触发场景 |
|---|---|
| 系统烧录 | *系统烧录* tab 启动烧录后 |
| OpenClaw 部署 / 升级 / 卸载 | *OpenClaw* tab 触发的长任务 |
| SFTP 批量传输 | *文件管理* 拖拽上传或下载多个文件 |
| 技能批量安装 | *技能工坊* 批量装技能 |
| 设备诊断 | *AI 对话* 触发的全面诊断 |
| Agent 编排（多步任务） | AI 委派给 OpenClaw 的板端任务 |
| 设备探测 | 添加设备时的连通性扫描 |

## 不进入顶栏的任务

任务队列只显示"工作台相关、前端可感知"的任务。以下任务**不**进入顶栏：

- 后台心跳（设备健康监控）
- AI 对话本身（在 AI Dock 中看进度）
- 单条 SSH 命令（在远程终端中看输出）
- WebSocket 长连接（在状态指示灯中看）

每个完整流程的进度、日志、操作（重试、暂停、取消）通常在对应功能页或对话内容中完成；只有符合"长任务 + 跨 tab 可感知"的才会以顶栏芯片形式出现。

## 操作

| 操作 | 怎么做 |
|---|---|
| 查看详情 | 点击芯片 → 跳转到对应功能页（如点烧录芯片跳到 *系统烧录* 页） |
| 取消 | 仅"Agent 编排"芯片支持取消按钮；其他任务的取消方式因功能而异 |
| 暂停 / 重试 | 不集中在顶栏，需要去对应功能页操作 |

## 标签页隔离

任务状态每浏览器标签独立维护：

- 关闭标签页 → 该标签的前端任务状态丢失
- 已发到后端或板端的命令是否继续执行 → 取决于各功能实现：
  - 烧录、SFTP：通常会继续完成（写到磁盘、远端）
  - SSH 命令：标签关闭后会话断开，但板端进程是否继续看具体命令（用 `nohup` 起的会继续）
- 关闭标签**不会**自动保证"终止"——重要任务建议保留标签直到任务真正完成

## 任务历史

顶栏只显示当前进行中的任务。查看历史的入口分散在各功能页：

| 任务类型 | 历史入口 |
|---|---|
| 烧录历史 | *系统烧录* tab → 顶部历史标签 |
| OpenClaw 操作历史 | *OpenClaw* → 操作日志 |
| AI 对话历史 | AI Dock → 会话列表 |
| SFTP 传输历史 | 当前未提供持久化历史，仅在标签页内有记录 |

这种分散设计的原因是不同任务类型的历史关注点差异较大：烧录关心"烧了哪些镜像"、OpenClaw 关心"做了哪些配置变更"、对话关心"上下文与回复"——把它们集中到一处反而难以查阅。
