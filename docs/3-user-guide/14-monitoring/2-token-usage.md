# 3.14.2 Token 用量统计

Token 是 LLM 处理文本的最小单位：英文 1 token 约等于 4 个字符（约 0.75 词），中文 1 token 约等于 1.5 个字符。模型计费基本都按 Token 数计算——Token 用量直接等于成本。

RDK Studio 在两处提供 Token 用量统计：AI Dock 顶部的会话胶囊与单次 Run 完成后的总结。本节同时说明当前版本的统计边界，避免开发者对统计能力的误判。

## AI Dock 顶部胶囊

AI Dock 顶部状态栏中的"用量胶囊"实时显示：

- 当前会话累计的 Token 数（输入 + 输出）
- 占当前模型上下文窗口上限的比例
- 接近上限时变红预警

胶囊数据是**会话级**的：切换到另一个会话时，胶囊显示该会话的累计；新建会话时归零。

## 单次 Run 完成后的总结

每次 AI 完成一轮回复后，会在消息尾部显示本轮 Token 用量概览：

| 字段 | 含义 |
|---|---|
| 本轮合计 | 输入 Token、输出 Token、总计（含思维链 Token，如有） |
| 轮次 | 本轮 Agent Loop 的迭代次数（工具调用后回转模型也算一次） |
| 工具调用次数 | 按工具名聚合，直观体现哪个工具贡献最多 |
| 耗时 | 从用户发送到 AI 完成的总时钟时间 |
| 折合费用 | 按模型厂商公布价格估算 |

Agent Loop 的典型流程：用户输入 → 模型决策（说话或调工具）→ 执行工具 → 把工具结果返回模型 → 模型继续决策 → 直到模型说"完成了"。一次用户消息可能引发多轮迭代，每轮都消耗 Token。

## 当前版本的统计边界

RDK Studio 1.1.x 的 Token 统计有明确的边界，**必须知道**：

| 你看到的 | 实际行为 |
|---|---|
| AI Dock 胶囊（本会话） | 准确，会话级实时 |
| Run 完成后的总结 | 准确，单 Run 累计 |
| 同一会话多轮加总 | 准确 |
| 跨会话累计 | **不持久化，重启 Studio 清零** |
| 按工具聚合的持久版本 | **不持久化** |
| 按日 / 周 / 月累计 | **不持久化** |
| 折合费用历史 | **不持久化** |

如果你的工作流需要严格的成本管控（如月度账单核对），不能依赖 Studio 内的累计数据。

## 严格成本管控的推荐做法

| 周期 | 推荐做法 |
|---|---|
| 短期（每日） | 每天截图 AI Dock 胶囊存档，便于对比 |
| 长期（每月） | 在模型厂商控制台开账单告警与日报，与 Studio 解耦的最可靠方式 |

模型厂商控制台的账单数据是真实计费数据，不会因 Studio 重启而丢失。

## 折合费用的精度

Studio 显示的折合费用是估算，可能与实际账单存在偏差，原因：

- 厂商实际计费可能包含输入缓存折扣、工具调用费、推理费等多项目
- 厂商价格调整时 Studio 可能未及时同步
- 批量折扣、企业合同价 Studio 不知道

要精确成本就看厂商账单。Studio 的费用估算只用作"日内大致比较"——例如对比某个任务比另一个贵多少倍。

## 实现层面

Token 统计在 Studio 进程内存里聚合：

- 计算入口：`server/monitoring/token-usage.ts`
- 流到前端：`src/hooks/useAIChatStore.ts` 的 `run-summary` 事件
- 工具：D-Moss Agent 的 `dmoss_token_usage_report` 工具可以查询当前进程内存中的累计数据
