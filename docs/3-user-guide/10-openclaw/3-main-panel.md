# 3.10.3 主面板与子页签

OpenClaw tab 的主面板由顶部状态卡片、6 个子页签、4 个全局按钮组成。

## 顶部状态卡片

状态卡片显示 OpenClaw 的关键运行信息：

- 运行状态（运行中 / 停止 / 异常）
- 当前版本号
- 监听端口（默认 18789）
- 当前使用的模型
- 飞书 / 微信通道的绑定状态

通过状态卡片可以一眼判断 OpenClaw 是否健康。

## 6 个子页签

| 子页签 | 内容 | 主要操作 |
|---|---|---|
| 对话 | 与板端 OpenClaw Agent 直接对话 | 不经过 PC D-Moss，直接让板端 Agent 处理 |
| 配置 | 模型条目、通道凭据、端口、自动安装策略 | 修改 OpenClaw 的运行参数 |
| 技能 | 板端已同步的技能列表 | 手动同步技能、卸载技能 |
| 配对 | 待审批的配对请求与已配对客户端 | 控制谁可以连接这个 OpenClaw |
| 日志 | journalctl 实时输出 | 排障、查看 OpenClaw 在做什么 |
| 监控 | CPU、内存、网关请求量、错误率 | 长期观察 OpenClaw 的资源消耗 |

### 对话子页签的特殊性

*对话* 子页签是直接与板端 OpenClaw Agent 对话，**不经过 PC 端的 D-Moss**。这种方式适用于：

- 让板端 Agent 自主处理任务，PC 不参与
- 测试板端 OpenClaw 的能力是否符合预期
- 在 PC 即将关机前，把任务交给板端 Agent 接管

注意板端 OpenClaw 的工具集与 D-Moss 不完全相同——板端工具更聚焦本地操作，缺少跨设备协调、PC 端文件操作等工具。

## 4 个全局按钮

| 按钮 | 用途 | 何时使用 |
|---|---|---|
| 设置 | 进入 *配置* 子页签 | 修改模型或通道配置 |
| 诊断 | 运行 `doctor` 全套检查 | OpenClaw 行为异常时的第一步排查 |
| 重启 | `systemctl --user restart openclaw` | OpenClaw 卡住时（类似手机 App 强制结束再开） |
| 升级 | 拉取最新版 npm 包重装 | 跟进 OpenClaw 上游版本 |

诊断按钮的输出包含 Node 版本、磁盘空间、监听端口、systemd 服务状态、网关连通性、模型 API 连通性等检查项，是定位问题的高效起点。
