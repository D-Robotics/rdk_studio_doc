# 3.6.1 启动与认证

## 自动启动流程

第一次打开 *远程桌面* tab，Studio 会执行以下步骤：

1. SSH 进板端检测是否安装了 `x11vnc`、`tigervnc` 或 `Xvfb`
2. 如未安装则弹出确认对话框，开发者确认后通过 `apt install` 安装
3. 如板端无显示器，自动启动 Xvfb 作为虚拟显示
4. 启动 VNC 服务监听本地 5900 端口
5. Studio 客户端通过 SSH 隧道在内嵌窗口连接 NoVNC 客户端

整个过程无需开发者手动操作。安装首次需要 1~3 分钟，之后打开是秒开。

## VNC 密码认证

VNC 服务支持 RFB 协议的密码认证：

- 第一次启动时 Studio 会让开发者设置 VNC 密码（也可生成随机密码）
- 密码加密存储在 *配置中心 → 设备连接* 的当前设备配置中
- 切换设备或在新机器上使用同一账号登录 Studio 时，密码自动同步，不需要重新输入

## 安全注意

VNC 端口 5900 不要暴露公网。RFB 协议历史上有多次安全漏洞，公网暴露风险较高。RDK Studio 默认通过 SSH 隧道访问，5900 端口仅监听板端本地 `127.0.0.1`，是安全的访问方式。

如需手动通过其他 VNC 客户端访问，建议：

- 在 SSH 客户端中建立端口转发：`ssh -L 5900:localhost:5900 root@<板端IP>`
- 然后用本地 VNC 客户端连接 `localhost:5900`
- 不要直接把板端 5900 端口绑定到 `0.0.0.0` 并暴露公网
