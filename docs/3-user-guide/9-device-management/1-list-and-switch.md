# 3.9.1 设备列表与切换

## 设备列表展示项

每台设备在列表中显示以下信息：

| 字段 | 说明 |
|---|---|
| 备注名 | 开发者自定义的设备名称 |
| IP 地址 | 当前接入使用的 IP |
| 在线状态 | 实时心跳结果（详见 [3.9.3](./3-online-monitoring.md)） |

列表支持按名称、IP、状态排序，方便在多设备场景下快速定位。

## 三种切换激活设备的方式

| 方式 | 路径 | 适用场景 |
|---|---|---|
| 顶栏左上角设备下拉框 | 任何 tab 都能切换 | 最快，日常使用首选 |
| 设备列表中的 *激活* 按钮 | *配置中心 → 设备连接* | 需要先看详情再决定切换的场景 |
| AI Dock 自然语言 | 直接说"切到 RDK-X5-工位2" | 编辑文档或代码时不愿离开当前位置的场景 |

无论哪种方式，切换后立即生效。

## 切换后的同步行为

| 项 | 切换后的行为 |
|---|---|
| 远程终端 | 默认在新标签中打开新设备的 SSH 会话；旧设备的标签保留 |
| 文件管理 | 自动指向新设备的根目录 |
| 远程 IDE | 切换到新设备的 code-server 实例 |
| 远程桌面 | 切换到新设备的 VNC 会话 |
| AI Dock | 切换到新设备对应的会话（每设备独立） |
| 工作台 | 重新加载新设备的指标数据（约 2~5 秒） |
| 顶栏 IP 列表 | 更新为新设备的所有网卡 IP |

切换时不会丢失旧设备的运行状态——已经在远程终端中跑的命令、已经在 AI Dock 中的对话历史都保留。下次切回旧设备时可以继续。

## 顶栏 IP 速查

切换设备后，点击顶栏的设备名会弹出该设备的所有网卡 IP：

```
RDK-X5-工位1
- wlan0: 192.168.1.45 (WiFi)
- eth0:  192.168.127.10 (网线直连)
- usb0:  192.168.128.10 (Type-C)
```

每个 IP 都可以一键复制——外部工具需要 IP 时不需要再 SSH 进板端查询。
