# 3.11.6 同步至板端

技能默认安装到 PC 端，由 D-Moss Agent 加载。如果希望板端常驻的 OpenClaw Agent 也能使用这些技能（例如长期监控、断网自治、在通道中处理消息），需要把技能同步到板端 OpenClaw 工作区。

## 同步流程

| 步骤 | 操作 |
|---|---|
| 1 | *技能工坊 → 已安装*（板端） |
| 2 | 选择需要同步的技能（也可勾选全部） |
| 3 | 点击 *同步技能* 按钮 |
| 4 | Studio 通过 SSH 隧道把指定技能传输到板端 OpenClaw 工作区 |
| 5 | 板端 OpenClaw 重新扫描技能目录，新技能立即可用 |

同步是单向的：PC → 板端。如果板端的技能与 PC 不同步，需要再次手动触发。

## 默认关闭自动同步

Studio 默认关闭"PC 安装新技能后自动同步到板端"。原因：

- 首次同步全部 45 个内置技能需要 1~3 分钟
- 不是所有技能都需要在板端使用（例如只在 PC 端运行的技能）
- 自动同步可能在用户不知情的情况下消耗带宽

如果团队工作流中确实需要"PC 与板端始终保持同步"，可以在 *OpenClaw → 配置* 中开启自动同步。

## 何时需要同步

| 场景 | 是否需要 |
|---|---|
| 板端只用作 SSH 远程目标，PC 始终在线 | 不需要，PC 端 D-Moss 加载技能即可 |
| 板端常驻 Agent 处理长期监控任务 | 需要，把监控相关技能同步到板端 |
| 微信 / 飞书 Bot 部署在板端 | 需要，把消息处理相关技能同步到板端 |
| 多板协作，希望每板都有相同能力 | 需要，每个板都同步一份 |

## 同步进度查看

同步任务在 OpenClaw tab 的 *日志* 子页签中实时显示：

```
[14:23:15] 开始同步 12 个技能到板端
[14:23:16] 推送 rdk-openclaw → /home/root/openclaw/skills/rdk-openclaw/
[14:23:17] 推送 rdk-device-ops → ...
...
[14:23:45] 同步完成，板端 OpenClaw 重新加载技能索引
```

如果同步中途失败（如网络断开），可以重新触发同步——已同步的技能会跳过，只补传缺失的。

## 板端技能的卸载

| 入口 | 操作 |
|---|---|
| Studio 内 | *技能工坊 → 已安装*（板端）→ 选中 → *从板端卸载* |
| 板端命令行 | SSH 进板端，删除 `~/openclaw/skills/<技能名>/` 目录 |

板端卸载只移除板端的副本，不影响 PC 端的同名技能。
