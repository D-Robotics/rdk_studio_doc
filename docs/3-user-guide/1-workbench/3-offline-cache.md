# 3.1.3 离线缓存机制

工作台在设备离线或网络抖动时不会一直转圈或显示空白，而是切换到低频心跳模式并展示缓存数据。这一机制让开发者在设备临时不可达时仍能保留对设备状态的认知。

## 离线判定与刷新策略

工作台的诊断轮询周期由 `DEVICE_POLL_PERIOD_MS` 控制（当前版本 **15 秒**），网络抖动或 SSH 不可达时 Studio 会在同一周期内自动重试：

| 设备状态 | 刷新行为 | 工作台显示 |
|---|---|---|
| 在线 | 每 15 秒一轮诊断与心跳 | 实时数据 |
| 响应超时 / 认证失败 | 下一轮自动重试 | 橙色诊断条提示具体原因 |
| 连续数轮失败（离线） | 继续以同样周期探测，等待恢复 | 顶部 Offline 标记 + 缓存数据 |
| 网络恢复 | 下一轮探测成功后即时刷新 | 橙条消失、数据变实时 |

设备恢复在线后，Studio 立即更新所有指标，无需手动刷新。

## 缓存数据的语义

缓存数据是设备最后一次在线时采集的指标快照，不是实时数据。这意味着：

- 工作台会标注数据时间戳（如"2 分钟前的状态"）
- 缓存数据可以帮助开发者回忆"设备失联前是什么状态"，但不能用于实时决策
- 关键场景（如温度监控）建议结合 [3.10 OpenClaw](../10-openclaw/index.md) 在板端常驻 Agent，断网时仍可继续监控

## 多设备时的工作台

工作台只显示当前激活的设备。切换设备的方法：

- 顶栏左上角的设备下拉框 → 选择另一台
- *设备管理* tab → 列表中的 *快速激活* 按钮
- AI Dock 直接说"切到 RDK-X5-移动"

切换后工作台数据完整重新加载（约 2~5 秒），不会显示前一台设备的旧数据。
