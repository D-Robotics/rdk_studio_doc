# 3.8.1 入口与状态显示

## 多个进入网络配置的入口

Studio 在多个位置都可以进入网络配置，全部使用同一套底层能力：

| 入口 | 适用场景 |
|---|---|
| 顶栏 WiFi 图标 | 已经接入设备，只想改 WiFi |
| 添加设备引导 → WiFi 步骤 | 首次接入板端，跟着引导走 |
| OpenClaw 部署 → WiFi 检查 | 部署 OpenClaw 时如果板端无网络，自动弹出网络配置 |
| 配置中心 → 设备连接 → 网络 | 高级设置入口 |

最常用的是顶栏 WiFi 图标，一键即可进入当前激活设备的 WiFi 配置。

## 顶栏 WiFi 状态实时显示

顶部工具栏显示当前激活设备的 WiFi 状态：

| 状态 | 显示 |
|---|---|
| 已连接 | WiFi 信号图标 + SSID 名称 |
| 未连接或 WiFi 关闭 | 灰色 WiFi 图标 + "未连接"提示 |
| 信号弱（&lt; -70 dBm） | WiFi 图标 + "信号弱"标记 |

点击图标弹出 WiFi 配置对话框。

## 实时连接日志

WiFi 配置弹窗底部会实时显示 `nmcli` 命令的执行日志：

```
[14:23:15] 扫描中...
[14:23:17] 找到 5 个可用网络
[14:23:23] 尝试连接 OfficeNet-5G ...
[14:23:24] 802.11 关联成功
[14:23:25] 4-Way 握手通过
[14:23:26] DHCP 获取 IP: 192.168.1.45
[14:23:26] 连接成功
```

如果连接失败，日志中会显示具体的失败步骤（信号弱、密码错、握手失败、DHCP 超时等）。把整段日志贴到 AI Dock，Agent 可以根据失败步骤给出针对性的解决方案。

## AI 替你换网络

也可以让 AI Agent 完成 WiFi 切换。在 AI Dock 中描述："板子换到 office_2.4g 这个网络，密码是 xxxx"。Agent 会断开当前连接、连接新网络、确认 IP 是否成功获取，整个过程结束后告知开发者结果。

如果切换的是另一台板（不是当前激活设备），可以在描述中明确指定：

> "把 RDK-X5-工位2 切到 lab_5g 这个 WiFi，密码 12345678"

Agent 会先切换到该设备的会话，再执行 WiFi 切换。
