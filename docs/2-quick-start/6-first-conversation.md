# 2.6 发起首次对话

至此，安装、登录、烧录、接入设备、配置网络、接入模型 6 步全部完成。本节带你给 AI 发出第一条消息，验证整个链路是否打通。

## 调起 AI Dock

AI Dock 是屏幕底部常驻的对话区，可以直接点击底部输入框激活。

## 第一条消息

在输入框中输入：

```
帮我查看这台设备的系统概况
```

按回车发送。Studio 会执行以下步骤：

1. 把消息发送给当前激活的 Thinking 车道模型
2. 模型理解意图后，自动调用设备工具（通过 SSH 在板端执行命令）
3. 几秒内返回当前激活设备的硬件指标：CPU、内存、磁盘、镜像版本、关键服务状态

整个过程中，Agent 调用的命令会在远程终端中实时显示，方便开发者了解 AI 在做什么。

## 几个值得尝试的初次对话

| 提示词 | Agent 会做什么 |
|---|---|
| `当前板上有哪些 ROS 节点在跑？` | 通过 SSH 执行 `ros2 node list`，返回节点列表 |
| `把 /tmp/test.txt 拷到我电脑桌面` | 调用文件传输工具，下载到 PC |
| `板上 BPU 占用率多少？` | 按板型执行 `hrut_bpuprofile -b 0`（X5）或读取 `/sys/devices/system/bpu/bpu0/ratio`（兜底），解析后返回实时数据 |
| `把 RDK X5 和 X3 的差异讲一下` | 调用内置硬件知识库回答 |
| `这个报错是什么意思：<粘贴报错>` | 匹配内置 30+ 种 RDK 错误模式，给出修复方法 |

## 几个使用建议

- **不要"开篇就让 AI 干大事"**：先问几个简单问题，让 Agent 了解当前板的状态
- **报错信息整段粘贴**：Studio 内置的错误模式识别依赖完整的报错原文
- **遇到工具调用确认时不要无脑同意**：默认 AI 调用破坏性命令（rm、kill 等）会先弹出确认窗口
- **不满意可以直接说**：例如"不要做 X，应该做 Y"，Agent 会调整策略

## AI Dock 的核心快捷键

| 操作 | 快捷键 |
|---|---|
| / | 光标定位到对话框 |
| 发送消息 | Enter |
| 多行输入换行 | Shift + Enter |


## 完整链路验证

如果第一条消息能正常返回设备状态，说明 RDK Studio 的完整链路已经打通：

- 桌面客户端（PC）
- D-Moss Agent（PC 端 AI 编排）
- 大模型 API（云端或自建）
- SSH 通道（PC ↔ 板端）
- 板端硬件响应
