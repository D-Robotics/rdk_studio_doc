# 5.11 模型回答质量问题

**典型现象**：模型回答跑题 / 重复 / 工具调用乱选 / 输出里出现 `<think>...</think>` 原文 / 中文回答夹杂英文（或反过来）。

## 30 秒决策

质量问题 90% 不是模型本身，先排除"环境干扰"：

1. 是不是 Quick 车道接管了原本要 Thinking 的任务？看 AI Dock 底部模型标签
2. 上下文使用率超过 90%？新建会话
3. 注入的技能太多互相干扰？在 AI Dock 输入 `/skills` 看清单
4. 真的是模型能力不行 → 换 Thinking 车道的模型

## 排查清单

1. **车道错配** — Quick 车道用 lite / mini 模型处理需要规划的任务，质量必然差。*AI Dock* 底部能看到当前实际使用的模型，不对就到 *设置面板 → AI 引擎* 把车道指派调对

2. **`<think>` 原文暴露** — 部分国产模型的 reasoning 模式会把推理包进 `<think>` 标签，Studio 的 InlineThinkingRouter 自动剥离。看到原文：
   - 升级到最新版（修复了多种厂商变体）
   - 或临时改 *Reasoning visibility* 为 `hidden`

3. **跨语言混杂** — *设置面板 → AI 引擎 → 高级* 把"回复语言"设为"跟随系统"或"中文"。极少数小模型仍偶发英文，换更强的模型最有效

4. **工具乱选** — *AI Dock* 输入 `/skills` 看注入了哪些技能。冗余技能太多干扰决策，就到 *技能工坊 → 已安装* 卸载用不上的

5. **历史对话拉胯** — 长会话累积太多上下文，模型表现下降。Studio 在 70% 时自动 compaction，但 90% + 还在追问就新建会话

## 永久解决

- Thinking 车道选最强可用模型，**不要为了省 Token 用 mini**
- 关键场景到 *技能工坊 → 创建新技能* 写一份**专门的 SKILL.md**，让 Agent 不需要"猜"该走哪条路
- 关注 RDK Studio 的版本升级提示，模型理解层（含 InlineThinkingRouter 等推理标签处理）会随版本不断改进
