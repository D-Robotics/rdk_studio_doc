# 5.9 多设备切换异常

**典型现象**：切换设备后 AI 对话上下文好像还停留在上一台 / 设备列表"丢了" / 同时连两台板，跑 `device_exec` 串了。

## 30 秒决策

*设置面板 → 设备连接* → 看"设备列表"：

- **列表为空** → 看下方"备份恢复"
- **列表完整** → 顶栏右上角设备下拉，确认当前激活设备

## 排查清单

1. **设备列表丢失** — 升级后偶发。*设置面板 → 设备连接 → 导入设备配置* 重新导入；没导出过的就重新填一次

2. **AI 对话上下文串了** — Studio 默认每个设备一个会话，切换设备会自动切到对应会话，但**模型 Token 用量是全局共享**。回复明显不是当前设备语境就 *AI 对话* 顶部 *新建会话* 强制开新窗口

3. **执行串到了别的设备** — AI Dock 顶部"目标设备"标签确认；通过 AI 自动选了"主设备"，在 AI 对话里直接说"对 X 设备执行 XX" 强制指定

4. **同设备并发限制** — Studio 给同一设备 SSH 并发上限默认 8，长任务挤占时其他探测会进等待队列**不是错误**，等正在跑的命令完成即可

## 永久解决

- 团队多人同板时约定**专人值守**，避免互相 kill 进程
- 给生产设备打**显著的 label**（添加设备时填"备注"字段）
- 定期 *设置面板 → 设备连接 → 导出设备配置* 备份
