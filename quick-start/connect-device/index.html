2.3 接入设备

RDK Studio 提供三种把板加入设备列表的方式。本节给出三种方式的对比与选择建议；详细操作步骤分别在三个子节中。

三种接入方式对比​

方式推荐场景优势限制
Type-C 接入板与 PC 在同一桌面，无路由器5 秒插上即可使用，无需配置 IP仅 RDK X5 支持
SSH 接入板已加入局域网（网线或 WiFi）通用、稳定、跨房间可用需要已知板的 IP
串口接入板系统启动失败或网络异常不依赖板端系统正常启动仅终端，无文件传输、远程桌面、IDE

选择建议​

你的硬件与板状态推荐方式
RDK X5 + 全功能 Type-C 数据线Type-C 接入（最快）
RDK X3 或 RDK S100SSH 接入（先把板接入路由器，从路由器后台或网络扫描获取板的 IP）
板系统损坏、SSH 无法连接串口接入（用于救火）

接入完成后的自动行为​

无论使用哪种方式，板加入 Studio 后会自动完成以下事项：

  • 顶栏的"当前设备"下拉框显示新增的设备，并自动激活该设备为当前操作目标
  • 远程终端 / 文件管理 / IDE / 远程桌面 自动指向当前激活的设备
  • AI 对话切换到该设备对应的会话（每设备独立会话）
  • 后台执行设备探测，识别板的型号、镜像版本、CPU/RAM、磁盘、TROS 状态等信息

设备识别的具体细节、多设备切换的高级用法见 3.9 设备管理。

通过 AI 接入​

也可以让 AI Agent 替你完成接入。在 AI Dock 中描述需求，例如：

  • "扫描一下 USB 上接的 RDK 板"——Agent 走 Type-C 接入流程
  • "我板子在 192.168.1.123，root/root，帮我加进设备列表"——Agent 走 SSH 接入流程
  • "板子启动崩了，帮我开个串口看 boot 日志"——Agent 走串口接入流程

AI Agent 会自动选择合适的方式、配置参数、建立连接，并把板加入设备列表。

后续操作​

完成接入后，建议给板配置 WiFi（2.4 配置网络），以便后续可以拔掉数据线或网线、通过无线网络远程访问。
